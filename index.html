How to showcase your Capstone Project

Showcasing your coding projects is an essential step towards presenting your skills as a Software Developer. It provides an opportunity to demonstrate your coding abilities, creativity, problem-solving, and communication skills. Here's a step-by-step guide on how to showcase your coding project effectively:

Step 2: Create a GitHub Account

If you have already created your account, then you can skip this step and proceed to Step 3.

GitHub is the most popular platform for hosting and sharing code repositories. Create a GitHub repository for your project and include all the necessary code files, including the README.md file.

How to Create a GitHub Repository

If you don’t have a GitHub account already, creating a GitHub account is a straightforward process that involves just a few steps. Here's how to create a GitHub account1:

  1. Go to the GitHub website: The first step is to navigate to the GitHub website at: GitHub .
  2. Sign up: Once you're on the website, click the "Sign up" button in the top right corner of the page.
  3. Enter your information: You'll be prompted to enter your name, a valid email address, and a strong password. You can also choose your username at this stage. Make sure to choose a username that's easy to remember and relevant to your interests.
  4. Choose your plan: GitHub offers both free and paid plans. If you're just starting with GitHub, the free plan. It provides enough features for most developers.
  5. Complete the verification process: After you've entered your information and chosen a plan, GitHub will ask you to verify your email address. Check your email inbox for a verification email from GitHub and follow the instructions to complete the process.
  6. Set up your profile: Once you've verified your email, you'll be taken to your GitHub profile page. Here, you can add information about yourself, including a profile picture, bio, and links to your website or social media profiles.

Step 4: Push your project to your GitHub account

To push a project to GitHub, follow these steps:

    • Go to GitHub.com and sign in to your account
    • Click the "+" button in the top-right corner of the screen and select "New repository"
    • Choose a name for your repository: You can choose a name like vending-machine-app for your module 1 capstone project and money-transfer-app for your module 2 capstone.
    • Make it public
    • DO NOT check the “Add a README file” as your project already has a README.md file
    • Click on “Create repository”
    • Once your project is created, you will see a window like below:
  2. Click on the "HTTPS" option and copy the URL provided by clicking the button as shown in the screenshot above
  3. Navigate to the directory where your project is saved on your local machine. If you are using MacOS or Linux, open your terminal and if you are using Microsoft Windows, open your GitBash and run the command " rm -rf .git " to delete your Git history.
  4. Create a new Git repository by running the following command: " git init "
  5. Add files to the staging area and commit your changes: In the terminal, run the command "git add ." to add all files in your project directory to the staging area. If you only want to add specific files, replace "." with the file names. Run the command "git commit -m '[Commit message]'" to commit your changes to the local repository. Replace [Commit message] with a short description of the changes you made (e.g. Initial commit).
  6. Connect your local repository to the remote repository: In the terminal, run the command "git remote add origin [Paste the GitHub repository URL you copied earlier]" to connect your local repository to the remote repository you just created on GitHub.
  7. Push changes to the remote repository: In the terminal, run the command "git push origin main" to push the changes from your local repository to the remote repository on GitHub.
  8. Check the remote repository: Refresh your browser and navigate to your repository on GitHub. You should see the files you just pushed. Congratulations! Your project is now successfully pushed to GitHub.

Step 5: Share your project:

After you push your project to your GitHub account, share it on various social media platforms, including Twitter and LinkedIn. You can also reach out to relevant communities and forums to showcase your project and get feedback from other developers.
